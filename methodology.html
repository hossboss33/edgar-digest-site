Methodology

How a filing becomes a note

Four steps, two of them mechanical. The verification step is ordinary string comparison, so you can reason about exactly what it does and does not catch.

1

Watch

A script polls SEC EDGAR each weekday for new filings from the watchlist, using the SEC's public submissions API. It costs nothing and touches no private data.

2

Draft

Claude, an AI model, drafts a short note for each new filing under one hard rule: every factual bullet must carry an exact quote from the filing text.

3

Verify

A separate program, not an AI, checks each quote against the filing character for character, and checks that every number in a bullet appears inside its own supporting quote.

4

Publish

Notes that pass every check publish here. Notes that fail any check go to a private review file and never publish.

What verification catches

Fabricated quotes. Altered quotes. Numbers that appear in a claim without appearing in the quote that supports it. These are the common failure modes of AI-drafted text, and string comparison catches them reliably.

What verification cannot catch

A quote can be accurate and still mislead out of context. A note can omit the most important fact in a filing. Long filings are truncated before drafting, so a note may reflect only the first part of a document. This is why every note links to its filing, and why you should read the source before acting on anything here.

Corrections

The pipeline republishes from source data on every run. If you find an error, check the linked filing first, then report it through the repository that hosts this site.

What this is

An automated research digest. An AI model drafts each note from the text of a public SEC filing. A separate program then checks every quote and figure against that filing and withholds anything that fails the check.

A verbatim quote can still mislead out of context, so read the primary source: every note links to the official filing on sec.gov. No warranty of accuracy or completeness.
